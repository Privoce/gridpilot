US interconnection map

Three roles. One filing bottleneck.

There is no national grid company in the US — generation and wires are private. Policy and markets are shaped by FERC and regional ISOs. Developers still assemble the application.

01 · Utility

The wire + retail operator

Who you pay for electricity. For smaller projects (often under a local MW threshold), the utility is the primary application counterparty — via portals like PowerClerk.

02 · Developer · our user

Builds the plant

Sites the project, runs studies, and files a thick interconnection application so the plant can connect — whether the offtake is the utility, a market, or behind-the-meter use.

03 · ISO / FERC

Market operator + regulator

ISOs (PJM, MISO, ERCOT, …) run the wholesale market and interconnection queues. Large / transmission-connected projects file into the ISO process; FERC sets the federal frame.

The expensive loop today

Developer → consultant (3–6 months) → file → utility/ISO review (another 3–6 months) → RFI → revise → pay again. Consultants do not guarantee acceptance. Every preventable SLD gap costs a queue cycle.

Where AI already shows up — and where it doesn’t

Tools are emerging to help utilities and ISOs review inbound filings faster. Almost nothing helps the developer run the same checklist before submission. That is GridPilot.

Product

Pre-filing co-pilot for the application packet

Start with the single-line diagram — the drawing reviewers fail first. Encode public utility and ISO requirements into rule packs, score the drawing, and clear blockers before PowerClerk or the ISO queue.

  1. 01 Pick utility + ISO path

    Distribution-scale → utility. Transmission-scale → ISO queue with the TO in the loop.

  2. 02 Upload the SLD

    Version CAD-exported PDFs the way your filing package will be reviewed.

  3. 03 Audit vs public rules

    Vision + rule packs from published interconnection requirements — not secret data.

  4. 04 Gate, then file

    Triage blockers, export a readiness report, then submit to utility or ISO.

Requirements differ by ISO, utility, and even county. GridPilot is built to ingest those public specs (Facilities Connection Requirements, tariff exhibits, portal checklists) into auditable rule packs.
